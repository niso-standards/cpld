A very simple example to showcase "packaging"

Introduction

It is not unusual for content to contain images, or even references to external media. This simple example shows how we can use the W3C Publication Manifest standard to create a content package for a Content Profile linked document.

This linked document is composed of:

  • This HTML file
  • Two JSON-LD @context files. One for CPLD, and one for W3C annotations
  • An image
  • A css file
  • Four web font files (the Atkinson Hyperlegible font, developed by the US Braille Institute)

This means that this paragraph can use a different font than the default.

The manifest.json file stitches everything together. As an example, it's using both a direct mention (of the example3.css file) and LinkedResource elements for the other elements.

The publication manifest standard allows the inclusion of a cover page, and other elements as well, but these were left out for brevity
